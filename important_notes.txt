BIOS:
What is a memory?
A: Memory is the piece of hardware used by computers to store information.

What is a bootloader?
A: Bootloader is a small computer program that loads the kernel of an operating system.

What is BIOS?
A:
1) Stands for Basic Input Output system.
2) It is a kernel in itself, it initializes hardware.
3) It looks for bootloader to boot by searching all storage mediums for the 0x55AA boot signature.
4) It is 16 bit code. Only 16 bit code can execute it properly.

Booting execution order: BIOS -> Bootloader -> kernel

Real Mode:
- Is a compatibility mode that all modern intel processors started when they are switched on.
- Can only excess 1 MB of RAM.
- Based on the original x86 design.
- No security.
- 16 bits accessible at one time.
- Memory access is done through segmentation memory model.
- We can only access 8 bit and 16 bit registers.

Segmentation Memory Model:
- Memory is addressed by segment and offset.
- Programs can be loaded in different areas of memory but run without problems.
- Multiple segments are available through the use of segment registers.

8086 Segment Registers:
- CS: Code Segment
- DS: Data Segment
- ES: Extra Segment
- SS: Stack Segment

Calculating Absolute Offset:
- Take the segment register multiply it by 16 and add the offset.
- Stack goes from high memory to low memory and heap goes from low memory to high memory. Stack grows downwards and heap grows upwards in x86.

Interrupts:
- Interrupts are like subroutines, don't need to know its memory address to invoke them.
- Interrupts are called through the interrupt numbers.

What happens when you invoke an interrupt?
- Processor is interrupted.
- Old state saved on stack.
- Interrupt executed.

Interrupt vector table explained
- Table describing 256 interrupt handlers.
- Entries contain 4 bytes (offset:segment)
- Interrupts are in numerical order in the table.

Interrupt Vector table:
- It is organized like this:
Offset (2 bytes) | Segment (2 bytes) | Offset | Segment | Offset | Segment | ... [256 times]
- Each interrupt is 4 bytes.
- Example: Interrupt 0 is at address 0x00
- Interrupt 1 at address 0x04
- Interrupt 2 at address 0x08
- Interrupt 0x13 at offset 0x13 * 0x04 = 76 (Decimal), processor looks at offset 76 in RAM:
  76-77 = Offset
  78-79 = Segment

Filesystem:
- Filesystems are kernel implemented they are not the responsibilty of the hard disk.
- Implementing a filesystem requires the kernel programmer to create a filesystem driver for the target filesystem.
- Data is read and written in sectors typically 512 byte blocks.
- Reading the sector of a disk will return 512 bytes of data for the chosen sector.
- Earlier sectors are read and written by specifying a "head", "track" and "sector".
- Currently, we use LBA (Logical block address) as a modern way of reading from a hard disk rather than specifying "head", "track" and "sector", we just specify a number that starts from zero.
- LBA allows us to read from disk as if we are reading blocks from a very large file.
- LBA 0 = first sector on the disk
- LBA 1 = second sector on the disk

Calculating LBA:
- Reading byte 58376: LBA = 58376/512 = 114
- Offset = 58376 % 512 = 8
- 114 * 512 = 58368
- 58368 + 8 = 58376

Protected Mode:
- Is a processor state in x86 architectures which gives access to memory protection, 4GB address space.
- Can provide memory and hardware protection.
- Different memory schemes.
- 4 GB of memory directly addressable.

Memory and Hardware Protection:
- Protected Mode allows you to protect memory from being accessed.
- Can prevent user programs talking with hardware.
- Ring 0: Kernel (Most privileged), Ring 1 and Ring 2 (Device drivers) (lesser privileged).
- Ring 3: Applications (Least privileged).

Selector Memory Scheme:
- Segmentation registers become selector registers.
- Selectors point to data structures that describe memory ranges and the permissions (ring level) 

Paging Memory Scheme
- Memory is virtual and what you address can point to somewhere entirely different in memory.
- Memory protection is easier to control.

4GB of Addressable Memory
- We gain access to 32-bit instructions and can easily work with 32-bit registers
- We can address up to 4GB of memory and no longer limited to 1 MB of memory provided by real mode.

Text mode:
- Allows you to write ascii to video memory. Ascii can be written into memory starting at address 0xB800 for coloured displays.
- Supports 16 unique colors. Each ascii character written to this memory has its pixel equivalent outputted to the monitor.
- No need to set individual screen pixels for printing characters.
- Each character takes up 2 bytes:
  1) Byte 0: Ascii character
  2) Byte 1: Colour code

Interrupt Descriptor Table:
- Describes how interrupts are invoked in protected mode.
- Can be mapped anywhere in memory.
- Different from interrupt vector table.
- Contains the following things:
  1) Offset (Bit 46-63) - Higher part pf the offset to execute
  2) P (Bit 47) - Should be set to xero for unused interrupts
  3) DPL (Bit 45-46, Descriptor Privilege Level) - The ring level the processor requires to call this interrupt   
  4) S (Bit 44, Storage Segment) - Should be set to zero for trap gates  
  5) Type (Bit 40-43, Gate Type) - The type of gate this interrupt is treated as  
  6) 0 (Bit 32-39, Ununsed 0-7) - Ununsed bits in this structure
  7) Selector (Bit 16-31, Selector 0-15) - The selector this interrupt is bounded to, the kernel code selector. 
  8) Offset (Bit 0-15, Offset 0-15) - The lower part of the offset to execute.

Print function semantics:
- void print -> terminal_writechar -> terminal_putchar -> terminal_makechar

Programmable interrupt controller (PIC):
- Allows hardware to interrupt the processor state.
- Programmable
- Requires interrupt acknowledgement.
- Allows different types of hardware to interrupt the processor such as hard disk, keyboard and more.
- Standard ISA (Instruction set architecture) IRQ's (Interrupt request):
  IRQ | Description
  0   | Timer Interrupt
  1   | Keyboard Interrupt
  2   | Cascade (used internally by the two PICs. never raised)
  3   | COM2 (if enabled)
  4   | COM1 (if enabled)
  5   | LPT2 (if enabled)
  6   | Floppy disk
  7   | LPT1
  8   | CMOS real time clock (if enabled)
  9   | Free for peripherals / legacy SCSI / NIC
  10  | Free for peripherals / SCSI / NIC 
  11  | Free for peripherals / SCSI / NIC 
  12  | PS2 Mouse
  13  | FPU / Coprocessor / Inter-processor
  14  | Primary ATA Hard Disk
  15  | Secondary ATA Hard Disk
